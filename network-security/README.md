# Secure Network Design — Merged Enterprise with Cloud Integration

## Project Overview

This project documents the design of a secure merged network for two acquired companies operating in regulated industries — financial services and healthcare technology. The merged organization required a unified network architecture that addressed compliance obligations across multiple regulatory frameworks while remaining within a defined first-year budget.

## What This Project Demonstrates

- **Vulnerability assessment** — Identified and analyzed existing vulnerabilities in both legacy networks, including impact, risk, and likelihood ratings
- **Network topology design** — Designed a merged on-premises and cloud hybrid architecture applying zero trust principles
- **OSI/TCP-IP mapping** — Identified the OSI model layer and TCP/IP stack layer for every component in the proposed topology
- **Regulatory compliance** — Addressed compliance requirements relevant to financial services and healthcare data environments
- **Budget-constrained decision making** — Justified every added, removed, or repurposed network component against cost-benefit tradeoffs within a $50,000 first-year budget
- **Emerging threat analysis** — Identified threats applicable to the merged organization post-integration, including performance and security risk management strategies
- **Professional recommendation** — Delivered a formal implementation recommendation with cost-benefit analysis for on-premises vs. cloud infrastructure

## Regulatory Frameworks Addressed

- **PCI DSS** — Payment Card Industry Data Security Standard (relevant to credit card processing in both companies)
- **HIPAA** — Health Insurance Portability and Accountability Act (relevant to medical software handled by the acquired company)

## Key Design Principles Applied

- **Zero Trust Architecture** — No implicit trust; every user and device is verified regardless of network location
- **Defense in Depth** — Layered security controls so no single failure exposes the entire network
